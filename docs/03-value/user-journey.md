# Teamo v1 — 核心用户路径

> Phase 3 | 创建时间: 2026-03-15 | 状态: 完成

## 核心用户路径（从首次访问到 Aha Moment）

```
[1] 打开 Teamo 首页
    ↓ 看到输入框 + "Claude Code 模式" 卡片
[2] 注册/登录（手机号/邮箱/Google）
    ↓ 获得初始 credits（赠送体验额度）
[3] 在输入框输入第一个任务 ← 关键转化点
    ↓ 例如："帮我用 Python 写一个 TODO API"
[4] 系统自动创建用户专属 Claude Code 实例（首次约 30-60s）
    ↓ 显示"正在为你启动专属 Claude Code..."
[5] Claude Code 开始执行，流式展示每一步操作 ← Aha Moment
    ↓ 思考 → 创建文件 → 写代码 → 运行测试
[6] 任务完成，展示结果（代码 diff、文件列表）
    ↓ 用户可以继续对话追问或下发新任务
[7] 关闭浏览器，实例持续运行
    ↓ 下次打开，项目上下文仍在
```

## 步骤详情

| 步骤 | 用户动作 | 系统响应 | 情绪 |
|------|---------|---------|------|
| 1. 首页 | 打开 teamo 网址 | 展示聊天界面，默认 Claude Code 模式 | 好奇 |
| 2. 注册 | 输入手机号/邮箱 | 发送验证码，完成注册，赠送 credits | 期待 |
| 3. 首次提问 | 输入任务描述 | "正在启动你的专属 Claude Code..." | 等待 |
| 4. 实例就绪 | 等待（30-60s） | 进度提示 + 实例状态变为"运行中" | 微焦虑 |
| 5. 流式执行 | 观看 AI 操作 | 实时展示思考、文件操作、代码编写 | **惊喜/Aha!** |
| 6. 结果展示 | 查看产出 | 代码高亮 + 文件树 + 可继续对话 | 满意 |
| 7. 持续使用 | 继续对话/关闭 | 实例保持运行，credits 按天扣除 | 信任建立 |

## 关键转化节点

1. **注册转化**: 首页看到效果 → 注册（需要降低注册门槛，考虑 visitor 模式试用）
2. **首次使用**: 注册 → 发送第一条消息（输入框预填 placeholder 引导）
3. **Aha Moment**: 看到 Claude Code 实时写代码的过程（必须流畅、可视化好）
4. **付费转化**: 免费 credits 用完 → 充值续费（体验足够好才会转化）

## 模式切换路径

```
默认: Claude Code 模式（新引擎）
  ↕ 一键切换
可选: Research 模式（旧引擎）
```

切换入口：输入框下方的模式卡片，点击即可切换。
切换时保持当前对话上下文，模式变更立即生效。
