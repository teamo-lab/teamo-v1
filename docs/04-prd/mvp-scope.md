# Teamo v1 MVP 范围定义

> Phase 4 | 创建时间: 2026-03-15

## MVP 模块清单

| 模块 | 优先级 | MVP | 工作量预估 | 依赖 |
|------|--------|-----|-----------|------|
| Claude Code 实例管理服务 | P0 | ✅ | 中 | deploy-claude-cloud |
| Claude Code 代理 API | P0 | ✅ | 中 | 实例管理服务 |
| 前端模式切换 | P0 | ✅ | 小 | 代理 API |
| Credits 计费 | P1 | ✅ | 小 | 实例管理服务 |
| 对话历史 | P1 | ✅ | 小 | 复用现有 |
| 用户系统 | P1 | ✅ | 无（复用） | 现有 wowchat |

## 不在 MVP 范围内

| 功能 | 优先级 | 原因 |
|------|--------|------|
| 多 Agent 协作 | P2 | 复杂度高，先做好单 Agent |
| 文件上传到实例 | P2 | MVP 通过对话描述需求即可 |
| 项目管理 | P2 | 先验证核心对话体验 |
| 团队协作 | P2 | 先做个人版 |
| 自定义 CLAUDE.md | P2 | MVP 使用默认配置 |
| 实例规格选择 | P2 | MVP 统一规格 |

## MVP 成功标准

1. **功能完整**: 用户可以通过 Web 界面向专属 Claude Code 实例发送任务并收到流式响应
2. **模式切换**: 可以在 Claude Code 和 Research 模式之间切换
3. **计费运作**: Credits 每日自动扣除，余额为 0 自动停机
4. **稳定性**: 连续 24h 无崩溃
5. **响应速度**: 首次实例创建 < 2 分钟，后续请求响应 < 3 秒

## 技术架构总览

```
┌──────────────┐     ┌──────────────────┐     ┌─────────────────────┐

│              │←────│  + Claude Code    │←────│ + Claude CLI 进程池   │
│              │     │    Proxy 模块     │ SSE │                     │
└──────────────┘     └──────────────────┘     └─────────────────────┘
                            │                         │
                     ┌──────┴──────┐           ┌──────┴──────┐
                     │  MongoDB    │           │  Redis      │
                     │  (历史/用户) │           │  (状态/缓存) │
                     └─────────────┘           └─────────────┘
```

## 开发顺序

1. **实例管理服务**（deploy-claude-cloud 二开）
   - 异步 Claude CLI 调用
   - 多用户实例管理
   - 流式输出 SSE

2. **Claude Code 代理模块**（wowchat-backend 新增）
   - mode 路由逻辑
   - SSE 格式转换
   - 对话历史保存

3. **前端模式切换**（teamo-frontend 改动）
   - ModeSelector 组件
   - chatcore.js mode 扩展
   - 实例状态展示

4. **Credits 计费**（wowchat-backend 扩展）
   - 每日扣费定时任务
   - 余额检查和停机
   - 前端余额展示
