# 关联产品实测报告

> Phase 2 | 创建时间: 2026-03-14 | 状态: 完成

## 产品 1: Devin (devin.ai)

### 注册流程
- 在 devin.ai 创建账号，完成 onboarding（约 1 小时）
- 连接 GitHub：Settings → Integrations → GitHub，OAuth 授权
- 选择 Devin 可访问的仓库，配置开发环境
- 起步价 $500/月（Core 计划），无免费层

### 核心工作流
```
用户发送自然语言任务（聊天/Slack/Linear）
    ↓
Devin 分析需求，制定计划
    ↓
在专属 VM 中执行：写代码、运行测试、调试
    ↓
创建 Git 分支，提交 PR
    ↓
用户审查 PR，合并
```

### 关键 UI/UX 模式
- **聊天式交互**：左侧聊天面板，右侧实时显示 Devin 的 IDE/终端/浏览器
- **任务列表视图**：可同时运行多个任务（最多 10 并发 session）
- **Slack 集成**：直接在 Slack 中 @Devin 下发任务
- **Wiki 自动生成**：自动为项目生成知识库

### 流式行为
- 实时显示 Devin 的操作步骤（文件编辑、命令执行、浏览器操作）
- 可以中途介入修改指令

### 实例模型
- 每个任务分配独立 VM（沙箱隔离）
- 任务完成后 VM 销毁
- 非持久化——每次新任务从头开始

### 值得借鉴
1. **Slack/Teams 集成**：降低使用门槛，不需要打开专门的 Web 页面
2. **自动 PR 创建**：任务完成直接生成可合并的 PR
3. **任务并行**：多任务同时执行，Dashboard 展示进度
4. **学习能力**：从项目代码模式中持续学习

### 体验问题
1. **价格过高**：$500/月起步，个人开发者难以承受
2. **质量不稳定**：复杂任务结果质量参差不齐
3. **需要详细 Spec**：模糊指令容易产出低质量结果
4. **黑盒模型**：用户不知道底层用什么模型，无法调优

## 产品 2: Claude Code (CLI + claude-code-webui)

### 使用方式
- 官方仅提供 CLI（终端命令行）
- Web 界面依赖第三方开源项目 [claude-code-webui](https://github.com/sugyan/claude-code-webui)
- 定价：Max $100/月（独立）或 $200/月（含 claude.ai Pro）

### 核心工作流
```
用户在终端输入自然语言指令
    ↓
Claude Code 分析当前项目上下文
    ↓
执行工具调用：读/写文件、运行命令、搜索代码
    ↓
流式输出每一步操作的详细过程
    ↓
用户确认或修改（human-in-the-loop）
```

### 关键交互模式
- **Status Line**：底部状态栏实时显示模型、成本、Git 分支、上下文使用率
- **Background Agents**：Ctrl+B 将子 Agent 放到后台，主会话继续工作
- **CLAUDE.md**：项目级指令文件，精确控制 Agent 行为
- **Skills/Hooks**：可扩展的技能和钩子系统
- **/tasks**：查看后台任务进度

### 流式行为
- 实时输出思考过程（thinking）、工具调用、工具结果
- 每一步都透明可见，用户可以中断
- SSE 流式输出（API 模式）

### 实例模型
- CLI 运行在用户本地——完全访问本地文件系统
- 云端版本（通过 Anthropic Console/API）支持远程执行
- 持久化：同一工作目录保持上下文

### 值得借鉴
1. **完全透明**：每一步操作都可见，用户有完全控制权
2. **CLAUDE.md 机制**：通过项目级配置精确控制 Agent 行为——Teamo 可以为每个用户自动生成
3. **Skills 系统**：可复用的任务模板，降低重复提示的成本
4. **上下文管理**：自动压缩、forking 等高级上下文管理

### 体验问题
1. **无官方 Web 界面**：只有 CLI，非技术用户难以使用——这正是 Teamo v1 要解决的
2. **跨会话记忆弱**：需要手动维护 memory 文件
3. **上下文窗口限制**：MCP 工具可占用 41% 窗口，实际可用空间受限
4. **API 成本高**：$200 订阅实际计算成本 ~$5,000，Anthropic 在大幅补贴

## 对 Teamo v1 的产品设计启示

| 维度 | 借鉴来源 | Teamo v1 设计方向 |
|------|---------|------------------|
| 任务提交 | Devin 的聊天式 + Slack 集成 | 保持 Teamo 现有聊天 UI，输入框直接发送任务 |
| 进度展示 | Claude Code 的透明步骤 + Devin 的并行任务 | SSE 流式展示每一步操作，支持多任务视图 |
| 控制能力 | Claude Code 的 CLAUDE.md | 为每个用户实例预配置 CLAUDE.md |
| 持久性 | Claude Code 的项目上下文 | 用户专属实例天然持久化，跨会话保持记忆 |
| 输出 | Devin 的自动 PR | 展示 Claude Code 的完整输出（代码 diff、文件变更） |
| 定价 | 避免 Devin 的黑盒 ACU | 28 credits/天透明计费 |
